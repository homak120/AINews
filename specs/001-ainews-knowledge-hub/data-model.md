# Data Model: AINews Knowledge Hub

**Branch**: `001-ainews-knowledge-hub` | **Date**: 2026-03-28

## Overview

Phase 1 has two data concerns:
1. **Static content** — `public/data/news.json` (read-only, generated by Claude Code trigger)
2. **User state** — `localStorage` (read-write, managed by the app)

No database. No server. All types defined in `src/types/index.ts`.

---

## Content Data (`data/news.json`)

### `NewsItem`

```typescript
type ContentType = 'video' | 'article' | 'paper' | 'social';

type Topic =
  | 'ai-engineering'
  | 'ai-research'
  | 'ai-career'
  | 'ai-industry';

interface KnowledgeCheck {
  question: string;
  options: string[];          // 3-4 choices
  correctIndex: number;       // 0-based index into options
  explanation: string;        // shown after answering
}

interface NewsItem {
  id: string;                 // URL-derived slug, e.g. "youtube-abc123"
  topics: Topic[];            // one or more topic tags
  type: ContentType;
  title: string;
  sourceUrl: string;          // original source URL
  youtubeId?: string;         // only present when type === 'video'
  publishedAt: string;        // ISO 8601 date string, e.g. "2026-03-25"
  summary: string;            // 2-4 sentence overview
  keyConcepts: string[];      // 3-5 concepts worth understanding
  whyItMatters: string;       // 1-2 sentence significance statement
  tags: string[];             // freeform keywords for future use
  relatedIds: string[];       // IDs of related NewsItems in same batch
  knowledgeChecks: KnowledgeCheck[]; // 2-3 questions; empty array if none
}
```

### `NewsData` (root shape of `news.json`)

```typescript
interface NewsData {
  generatedAt: string;        // ISO 8601 timestamp of content generation
  weekOf: string;             // ISO 8601 date — start of the 7-day window
  items: NewsItem[];
}
```

**Validation rules**:
- `id` MUST be unique within the file
- `relatedIds` MUST only reference IDs that exist in the same `items` array
- `youtubeId` MUST be absent (or omitted) when `type !== 'video'`
- `knowledgeChecks[].correctIndex` MUST be a valid index into `options`
- `topics` MUST contain at least one entry

## User State (`localStorage`)

All user state is stored under a single namespaced key: `ainews:preferences`.

### `UserPreferences`

```typescript
interface KnowledgeCheckResult {
  completedAt: string;        // ISO 8601 timestamp
  answers: number[];          // selected option index per question
}

interface UserPreferences {
  bookmarks: string[];                          // ordered list of item IDs
  notes: Record<string, string>;                // itemId → note text
  knowledgeCheckResults: Record<string, KnowledgeCheckResult>; // itemId → result
}
```

**Default value** (when localStorage key is absent):

```typescript
const DEFAULT_PREFERENCES: UserPreferences = {
  bookmarks: [],
  notes: {},
  knowledgeCheckResults: {},
};
```

**Persistence rules**:
- Written on every change (debounced for notes to avoid excessive writes)
- Read once on app init; held in Context for the session
- Never cleared automatically; user must explicitly remove bookmarks/notes

---

## State Architecture

```
NewsContext          — holds NewsData (loaded from news.json via Axios)
UserPreferencesContext — holds UserPreferences (synced to localStorage)
```

```
useNews()             → { items, getById, getByTopic, isLoading, error }
useBookmarks()        → { bookmarks, toggle, isBookmarked }
useNotes()            → { getNote, setNote }
useKnowledgeChecks()  → { getResult, saveResult, isCompleted }
```

Each hook reads from the relevant Context — no prop drilling, no direct
localStorage calls outside the `useLocalStorage` base hook.
