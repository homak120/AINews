# CLAUDE.md — AINews

## Project Overview

AINews is a personal AI knowledge hub — a weekly-refreshed single-page
application that replaces scattered YouTube searches and web browsing with one
organized, educational feed of AI news.

**Current phase**: Phase 1 MVP (static JSON data source)
**Deployment target**: Self-hosted → Vercel

## Key Files to Read First

Before starting any work, read these in order:

1. `.specify/memory/constitution.md` — non-negotiable principles, tech stack,
   deployment strategy. This supersedes everything else.
2. `specs/001-ainews-knowledge-hub/spec.md` — active feature specification
   with all user stories, requirements, and clarifications.
3. `docs/ux-design-guide.md` — **mandatory before any UI implementation** —
   complete visual design spec: color tokens, typography, component JSX classes,
   interaction rules, and Tailwind config extension. All UI must match this.
4. `README.md` — product overview and phased delivery plan.

## Non-Negotiables (from Constitution)

- **TypeScript strict mode everywhere** — no `any`, no plain `.js` files in `src/`
- **React + React Router only** — no other UI frameworks
- **Tailwind CSS only** — no custom CSS files, no external component libraries
- **State: useState / useReducer / Context API only** — no Redux, Zustand, or
  other state libraries
- **Phase 1 reads `data/news.json` only** — no live external API calls
- **Axios** is available but Phase 1 use is limited to loading local JSON
- **localStorage** for all user persistence (bookmarks, notes, knowledge checks)
- **YAGNI** — do not build Phase 2 abstractions during Phase 1

## Content & Data

- **Topics**: AI Engineering & Skill Development · AI Research & Future
  Technology · AI Career & Workforce Intelligence · AI Industry Strategy &
  Product Landscape
- **Item ID format**: URL-derived slug (e.g., `youtube-abc123`, `arxiv-2401-12345`)
- **Default view**: All 4 topic sections shown simultaneously (dashboard layout)
- **Content generation**: Claude Code scheduled trigger with web search,
  auto-generates summaries, key concepts, why-it-matters, knowledge checks,
  and related item links. Operator reviews before deploying.
- **Content generation prompt**: `specs/001-ainews-knowledge-hub/contracts/content-generation-prompt.md`
  — authoritative prompt template, tone guide, and validation checklist.

## Behavior Guidelines

- Always check the constitution before making architectural decisions
- Do not add dependencies not listed in the constitution without flagging it
- Do not implement Phase 2 features (live API, auth, scheduling) in Phase 1
- When in doubt between two approaches, choose the simpler one (YAGNI)
- Mobile responsiveness is desirable but not a hard requirement for MVP
